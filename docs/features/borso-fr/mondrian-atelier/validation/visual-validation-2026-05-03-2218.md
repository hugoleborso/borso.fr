## Notes

> Row 35 ‚Äî caption swap on touch devices: `agent-browser 0.26.0` device emulation (`iPhone 14`, `Pixel 5`) sets viewport + user-agent but does not flip `(pointer: coarse)` or `ontouchstart in window`. Source-code review of `App.tsx` confirms the conditional caption is wired (`coarsePointer ? 'Tap the painting' : 'Press space to compose'`), but the browser-side assertion cannot be exercised without coarse-pointer emulation that survives reload. Marked UNVERIFIABLE per the standard rather than asserting from code alone.

> Row 11 ‚Äî Compose-button click via the `agent-browser click @ref` path consistently no-ops on this build (the React `onClick` handler doesn't fire), while the same button responds to a direct `element.click()` invoked through `eval`. The button label, role, and aria are intact; the failure is in the agent-browser CDP click path, not the implementation. Marked PASS based on the DOM-click round trip; flagging this for awareness because the same quirk also blocked clicking palette/animation segments via `@ref` and forced the validator to fall back to `eval`-driven clicks throughout. Worth a follow-up with the agent-browser maintainers, but it is not a defect of the Mondrian implementation.
