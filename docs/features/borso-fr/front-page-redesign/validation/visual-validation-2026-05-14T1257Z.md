## Notes

> One bullet per FAIL or UNVERIFIABLE row.

- **#12 (Mobile menu fills screen) — FAIL.** Spec says: *"Mobile (< 640 px) : menu prend tout l'écran (centré, backdrop-blur), font-size 22 px."* At viewport 360×640, the `.menu` measured `width: 286.4 px, left: 73.59 px, right: 0 px`. The mobile media query in `apps/borso-fr/site/style.css` (lines 173–186) sets `padding: 0 24px; align-items: center; justify-content: center; background: rgba(5,7,13,0.65); backdrop-filter: blur(12px)` but never overrides the desktop rule's missing `left` — `.menu { position: fixed; top: 0; right: 0; bottom: 0; }` has no `left`, so the element width is intrinsic. The screenshot `04-mobile-menu-open.png` shows a visible ~74-px strip of the galaxy uncovered on the left side, including a partial "b" of the title bleeding through. Two CSS-only fixes possible: (a) add `left: 0` to the base `.menu` rule, or (b) set `left: 0` inside the `@media (max-width: 640px)` block. The backdrop-blur, centering, and 22 px font-size are all correctly implemented — the bug is solely the missing horizontal stretch.
