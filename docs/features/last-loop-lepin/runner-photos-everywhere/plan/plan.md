# Plan — runner photos everywhere

> Early quality check. Pair with [`../spec/spec.md`](../spec/spec.md). When a defect lands and a Dantotsu traces back here, the chain is visible: the plan either named the risk and we missed mitigating it, didn't name the risk at all (planning gap), or named it correctly and the defect comes from elsewhere.

The four avatar surfaces (`Leaderboard` chips, `CourseMap` Leaflet markers, the sister `ElevationProfile` pastilles, `SelfPunchModal`) today render `initialsAvatar(displayName)`. The spec plumbs `runner.photoKey` (already uploaded, never displayed) through to those surfaces via a server-computed `photoUrl` and a new `<RunnerAvatar>` component with an `<img onError>` cascade to initials. The infra slice (S3 photos bucket → CloudFront → DNS), the upload-time Lambda resize, the DTO mapper, and the React/Leaflet integration are all in scope.

### Pattern Coherence pass

- **Image resize library.** No image-processing dep in the repo today (`grep "sharp\|jimp\|@aws-sdk/client-s3" apps/last-loop-lepin/api/package.json` shows only `@aws-sdk/client-s3`). Picking `sharp` adds the `node_modules` heaviest single dep (~30 MB unpacked, native libvips). The Lambda runs at upload time only — not request-path — so cold-start cost is irrelevant for the API. `sharp` is the default for S3 resize Lambdas across the AWS docs; alternatives (`@squoosh/lib`, hand-rolled WASM) carry their own footguns. **Decision: `sharp`**, but track a follow-up ADR if bundle size becomes a Lambda-layer pain (see §A.2 in Risk register and Open questions).
- **Server-side DTO mapping.** Today the API returns domain types directly (`context.json({ runner })`). Introducing one mapper for `Runner → RunnerDto` is a new pattern. **Decision: mapper lives in `apps/last-loop-lepin/api/src/runner/runner.dto.utils.ts`** as a pure helper (deterministic in `Runner` + `env.PHOTOS_CDN_HOST`), 100% covered. Used by every controller that currently emits a `Runner`. No global `RankedRunner → RankedRunnerDto` machinery — keep this mapper-per-domain-leaf so it scales without becoming a "DTO layer" framework.
- **React state on image-error cascade.** `useEffect` ban from CLAUDE.md is in force. The cascade is an event handler (`<img onError>` is a DOM event), not a synchronisation with an external system — handled with `useState` set inside the handler. No `useEffect`. See §E.

| **§A.2 — `infra/cdk/` 100% coverage gate** breaks if the new construct ships untested branches.
