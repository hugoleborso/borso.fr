## Notes

- **C05 UNVERIFIABLE — back-e2e tests not run.** The validator brief explicitly forbade running `pnpm run test` because three validator worktrees would race on the shared local-postgres cluster. The implementing agent reported them green in the implementation thread. Operator must echo the impl-agent's evidence in the PR description per the disclosure rule.
- **A16 minor signature divergence.** The spec wrote the signature as `registerSelfPunch(db, input, edition, userAgent, now)` (edition pre-loaded by the caller). The implementation is `registerSelfPunch(db, input, userAgent, now)` and loads the edition inside the function via `getEdition`. Behaviourally equivalent — both paths read the edition before computing distance — and consistent with the sibling `registerPunch` shape. Calling it PASS rather than dragging it through a "spec drift" lens; flagging here so the operator sees it.
- **Plan-vs-impl drift on z-validator + missing User-Agent.** The plan's `self-check` line for `self-punch.controller.ts` says "Test back-e2e : … 400 sans User-Agent header (z-validator)." The implementation does NOT reject missing UA — the controller falls back to `null` (`context.req.header(USER_AGENT_HEADER) ?? null`) and writes `userAgent: null`. The spec doesn't require rejection; the plan note is a planning slip, not a spec violation. No verdict change.
- **Plan checkbox "tsconfig.json: pas de changement"** — confirmed, no `tsconfig.json` change in the diff.
- **R-DSQL** dry-run was a human gate at plan time (gate 7); not re-runnable in this validator's sandbox. The migration as shipped is the single-statement form (`ADD COLUMN … NOT NULL DEFAULT 'admin'`); if the human dry-run had failed the impl would have switched to the 3-statement fallback. Trust delegated to the impl agent's gate 7.
- All 9 risks named in the plan map to a concrete mitigation visible in the diff (R-DSQL: comment + fallback documented in migration; R-FSM: `assertNever`; R-IP-leak: no `ip` column / no controller IP read; R-Acc-100m-edge: shared `GEOFENCE_RADIUS_METERS` + shared test vector at the boundary; R-Geo-timeout: 10 s timeout + FSM `timeout` state; R-DOM-button: `type="button"`, classes preserved, fall-through to `<div>` on retransmission; R-Drift-haversine: byte-identical files + shared test vectors; R-Geofence-startLatLng-mauvais: 100 m radius, smoke-test gate delegated to operator; R-Couverture-cellulaire: admin remains; R-RTL-tests-front: FSM extracted to `*.utils.ts`, shell delegated to `/visual-validation`).
