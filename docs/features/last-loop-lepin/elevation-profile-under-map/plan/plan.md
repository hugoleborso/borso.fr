## Pattern coherence pass

- **Projection of a runner onto the loop** — pre-feature, this lives inline in `CourseMap.tsx`'s second `useEffect` (lines ~96–155). The new component needs the same value, so we *do not* duplicate the inline arithmetic — we **extract it** into a single pure helper `runnerDistanceFraction(edition, entry, now)` in `course-map.utils.ts`, called by both `CourseMap.tsx` and `ElevationProfile.tsx`. Same shape as the previous `gpx-speed-aware-projection` PR which already extracted `projectFraction` / `projectFractionTimeAware`. One projection rule, one home.
- **Optional-array on `trackJson`** — the prior `pointTimeFractions` key established the convention: `null` at the back core boundary, `undefined` (omitted JSON key) at the wire boundary, conversion centralised in `edition.service.ts.trackJsonOf`. `pointElevations` follows the same pattern verbatim — no second pattern introduced. Pure widening of the existing one.
- **Pure helpers** — both new modules end in `*.utils.ts` (`elevation-profile.utils.ts` and the extended `course-map.utils.ts`), gated at 100 % by the existing workspace rule `site/src/**/*.utils.ts`. No "*Component*.tsx with inline algorithm" pattern reintroduced.
- **No new dep** — SVG is JSX standard. No chart library (Recharts / ECharts / uPlot / Chart.js) joins `package.json`. Audit of imported deps: no port-in from this feature.
