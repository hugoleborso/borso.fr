## Open questions / unknowns

**Â§A. `null` vs `undefined` at the core/DTO boundary.**
Spec lines 88 and 99 disagree: `GpxMetadata.trackJson.pointTimeFractions?: ReadonlyArray<number>` (optional, undefined) vs `GpxTrack.pointTimeFractions: readonly number[] \| null`. Resolution adopted in this plan: **keep both as-written** â€” `null` is the core-boundary signal that "we parsed but there was no series" (distinct from "we haven't parsed yet"), `undefined` is the wire-boundary signal that the JSON key is absent. Conversion is centralised in `edition.service.ts` (spread-when-non-null). If the implementation surfaces friction, the fallback is to use `undefined` on both sides and drop the optionality from the parser's return type. **No blocker.**

**Â§B. `__fixtures__` folder does not exist in the repo.**
Spec line 138 references `apps/last-loop-lepin/api/src/helpers/gpx/__fixtures__/strava-recorded.gpx`. No `__fixtures__` directory exists anywhere in `apps/last-loop-lepin/` today; existing GPX test vectors are inline string constants (`MINIMAL_GPX` in `gpx.core.test.ts`). Resolution adopted: **inline the sub-sampled GPX as a string constant**, matching the existing pattern. No new directory pattern introduced. **No blocker.**
