# Plan — Last Loop Lépin · race day live

> Early quality check. Pair with [`../spec/spec.md`](../spec/spec.md) (commit `43c61ae`) and the Claude Design hand-off at [`../spec/mockups/`](../spec/mockups/). When a defect lands and a Dantotsu traces back here, the chain is visible: the plan either named the risk and we missed mitigating it, didn't name the risk at all (planning gap), or named it correctly and the defect comes from elsewhere.

**Scope v1 (confirmed 2026-05-12, post back-end discussion).** Full-stack first-of-its-kind in this repo: Vite + React `site/`, **Hono on Lambda** behind `LambdaApi`'s API Gateway HTTP API, **Drizzle ORM + postgres-js + `@aws-sdk/dsql-signer`** against the per-app DSQL cluster, CDK deploy via `PreviewableApp`. The Claude Design prototype under `spec/mockups/` is the visual target; we recreate its surfaces in React + the repo's CSS variables, never copy its prototype scaffolding (`__edit_mode_*`, tweak-panel, `data-aesthetic` runtime toggle).

## Pattern Coherence pass

`apps/last-loop-lepin/` is brand-new — nothing to inherit. But this app is **the first consumer in the repo of**:

- A **back-end with code** (the two existing apps are frontend-only). Every choice we make here (Hono, Drizzle, testcontainers, two-gate vitest, no-abbreviation naming, no `kernel/` / `platform/` dumping ground) becomes the seed of a back-end pattern other future apps will copy. The plan therefore commits explicitly to a small, principled API surface — not "whatever Hono does by default", not "whatever Drizzle's quickstart shows".
- `PreviewableApp` (the composite construct `StaticSite + LambdaApi + DsqlSchema`). Construct exists but has zero production miles. The risk register treats this as a high-severity landing-strip risk with explicit mitigation (CDK template assertions + early preview deploy).
- `DsqlSchema` migrating an actual schema beyond the trivial test fixtures. Same caveat.

The two existing apps both use plain `useState` (no zustand, no Redux, no state machine library) and `useSyncExternalStore` for clock-like external state. We stay on that — no new state-management dependency. The Hono dependency lands here intentionally; `borsouvertures` and `borso-fr` stay frontend-only.

The Claude Design mockups carry three aesthetics (scoreboard / topographic / telemetry) and a runtime "tweak panel" for design exploration. The plan ships **one** aesthetic (`scoreboard`) and drops the tweak panel entirely — they are design-tool affordances, not product.

## Risk register

| Risk | Severity | Mitigation in plan | Detection if it slips |
|---|---|---|---|

| **Premier consumer de `DsqlSchema` avec un schéma réel** — au-delà des fixtures de test | **high** | Les migrations `0001_initial.sql` sont générées par `drizzle-kit generate` et appliquées **à l'identique** côté testcontainers (via `drizzle-kit migrate`, pas `push`) ET côté prod (via le construct). ## Open questions / unknowns

- **`awslambda.streamify` vs `LambdaApi` via API Gateway HTTP API.** Spec mentionne le streaming ; le construct existant ne le supporte pas. Plan choisit de ship sans streaming en v1. Si l'utilisateur veut le streaming, c'est un PR séparé qui refactor `LambdaApi` vers Function URL — flag à confirmer.
- **`drizzle-kit push` vs `drizzle-kit migrate` côté tests.** Spec dit "schéma rejoué via `drizzle-kit push`". Plan dévie : `migrate` est plus fidèle au chemin prod (`DsqlSchema` applique les fichiers `.sql` committés). Tradeoff : `push` est plus rapide (~200 ms) mais cache les bugs de migration. Plan recommande `migrate` ; à confirmer.
- **Nom du package npm.** Convention existante = `@borso-app/<slug>` (les deux apps existantes). Spec utilise `@borso/last-loop-lepin` une fois (par mégarde, ailleurs c'est cohérent). Plan use `@borso-app/last-loop-lepin`.
- **PIN storage.** Plan = Argon2id hashé dans `PIN_HASH` env var, posée par `cdk.json` ou Secrets Manager. Alternative = SSM SecureString fetché au cold start (~80 ms penalty). Plan default = env var, simpler.
- **Idempotency-key column dans `loop_punches`** — la spec mentionne idempotency dans l'edge case 4G mais ne nomme pas la colonne. Plan ajoute `idempotency_key TEXT NULL` et une contrainte unique partielle. À confirmer si l'utilisateur préfère un UUID natif ou un hash dérivé.
- **`Cache-Control` exact sur `GET /api/editions/:slug/state`.** Plan = `max-age=2, stale-while-revalidate=10`. L'input metric `loop_punched p90 < 2 s` se satisfait de cette valeur mais on pourrait baisser à 1 s + 5 s. À éprouver au dry-run J-7.
- **`postgres-js` lazy password callback** — drizzle-orm + postgres-js supportent une fonction async pour `password` ; vérifier qu'AWS signer s'intègre proprement sans bouclage. Si non, fallback = manual token refresh dans un `process.nextTick` setup.
- **Sentry vs Plausible/Posthog** pour les 5 analytics events. Plan = breadcrumbs Sentry (pas de nouveau vendor). Si product veut des funnels, c'est un PR follow-up.
